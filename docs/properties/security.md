# Security invariants

Cross-crate security contracts: credential redaction on builder and env-override surfaces, non-canonical host rejection by default, and secret-shaped payload redaction across every reviewed public error family. Part of the [Properties Registry](index.md): 3 invariant(s), 3 covered.

| `PROP-SEC-003` | whole workspace | Every reviewed public error family redacts or safely types secret-shaped payloads before those values reach `Display`, `Debug`, or JSON serialization through the SDK facade, and the `redact_response_body` scanner enforces a layered JWT, Bearer, strict-URL, bare-userinfo, and credential-keyed-value detector pipeline with recursive key-prefix scanning so partner response echoes cannot bypass the credential firewall through scheme mangling, mid-word Bearer tokens, JWTs embedded in credential keys, or partial credential-key spellings. App-data document validation surfaces failures as typed `AppDataError` values whose `Display` names only the offending public field and the canonical `ValidationReason`, never the caller-supplied value.
