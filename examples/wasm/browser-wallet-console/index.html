cow-rs Wallet Console

Sign, submit, and cancel CoW Protocol orders in a browser with a deterministic mock wallet or a real injected EIP-1193 provider.

This console keeps deterministic and wallet-backed browser proof separate. Use Mock Wallet for a no-extension async flow through the public SDK, and Injected Wallet for live window.ethereum connect, sign, quote, submit, and cancel actions.

Run Mode

Static browser-live orderbook actions are enabled on this page only for staging. Production requires a proxy-enabled deployment instead of the default static serving path.

Injected Wallet

Live wallet path for connect, sign, quote, submit, and cancel through the root `cow-sdk` surface. Detect caches reviewed wallet candidates, Confirm Wallet is only required when more than one candidate is present, Connect / Reconnect uses the confirmed provider or the retained wallet handle, Switch Chain must align the wallet session with the selected console chain before live actions, Reset Session keeps the selected wallet, and Clear Console Wallet clears retained console state without revoking extension authorization. Static browser-live orderbook actions are enabled here only on staging; production requires a proxy-enabled deployment.
